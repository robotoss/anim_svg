# anim_svg_core

Native (Rust) SVG → Lottie converter for the `anim_svg` Flutter package.

Phase 1 scaffold: the FFI boundary and build system are in place but the
converter pipeline is still a stub. Each `anim_svg_convert` call returns
an envelope with a `unsupported_feature` error. Real parsers/mappers land
in later phases.

## Public C API

See `include/anim_svg_core.h` (generated by `cbindgen` on each build):

- `anim_svg_convert(svg, opts) -> const char*` — returns a malloc'd JSON
  envelope: `{ lottie, svg_raw, logs, error }`. Caller frees with
  `anim_svg_free_string`.
- `anim_svg_free_string(ptr)` — frees any string returned by this API.
- `anim_svg_core_version() -> const char*` — static version string.

## Toolchain requirements

- Rust stable (>= 1.80 recommended).
- For iOS: Xcode command-line tools (`lipo`, `xcodebuild`). Rustup targets
  `aarch64-apple-ios`, `aarch64-apple-ios-sim`, `x86_64-apple-ios-sim`.
- For Android: [`cargo-ndk`](https://github.com/bbqsrc/cargo-ndk)
  (`cargo install cargo-ndk`) and an Android NDK r25+. Rustup targets
  `aarch64-linux-android`, `armv7-linux-androideabi`,
  `x86_64-linux-android`, `i686-linux-android`.

## Build

From the package root:

```
./tool/build_rust.sh        # iOS + Android (macOS host) or Android only (Linux)
./tool/build_rust.sh ios
./tool/build_rust.sh android
```

Artifacts:

- `ios/Frameworks/anim_svg_core.xcframework`
- `android/src/main/jniLibs/<abi>/libanim_svg_core.so`

`pod install` and `./gradlew build` will call `tool/prepare_rust.sh`
automatically when artifacts are missing. Set `FORCE_RUST_REBUILD=1` to
force a rebuild.

## Test

```
cd native/anim_svg_core
cargo test
```

And from the Flutter package (requires local dylib from `cargo build`):

```
flutter test test/ffi/rust_converter_smoke_test.dart
```
